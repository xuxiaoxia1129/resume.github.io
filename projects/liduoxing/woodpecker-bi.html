股民保护 BI 看板 ​

股民保护 BI 看板是围绕投资者权益维护业务搭建的数据产品项目。它把 公域曝光、资源进线、私域咨询、客户分类、签约转化、维权标的和法务流程 组织成一条可追踪的数据链路，帮助业务从手工报表升级为可复盘的经营看板。

2025 年启动股民保护与投资者权益维护业务项目
渠道漏斗公域曝光、私域进线、咨询、签约、立案
看板原型总览、渠道、运营多类 dashboard demo
数据口径资源、A 类客户、维权股、标的金额、营收估算

项目背景 ​

股民保护项目围绕投资者权益维护业务展开，业务通过公域内容、视频号、公众号、私域 H5、客服咨询和法务流程承接潜在客户。

早期数据主要依赖人工维护和定期汇报，管理层难以持续追踪：

  • 公域账号带来了多少曝光和粉丝？
  • 哪些渠道带来的资源更有效？
  • 客户从私信咨询到签约转化的路径是什么？
  • 哪些客户符合维权条件？
  • 哪些维权标的贡献了主要签约和金额？
  • 手工报表中的资源数、A 类客户、营收估算等口径是否统一？

因此，项目需要将业务流程线上化、指标化和看板化。

项目目标 ​

  1. 梳理股民保护业务流程，从公域曝光到签约和后续维权形成完整路径。
  2. 明确资源、客户分类、维权股、标的金额、营收估算等关键口径。
  3. 建立管理层驾驶舱和运营看板，持续追踪渠道效率和签约转化。
  4. 将人工报表和业务经验转化为可开发、可验收的数据产品需求。
  5. 为后续线上化和自动化采集打基础。

我的角色 ​

我负责业务理解、流程图梳理、数据源口径整理、BI 看板需求和原型 demo 推进。这个项目的重点，是先理解业务方真正想追踪的路径，再把报表拆成可开发的数据对象。

工作方向我承担的工作
业务梳理理解公域账号、私域咨询、客服承接、客户分类、签约和法务流程
口径确认梳理资源数、A 类客户、维权股、标的金额、营收估算等概念

交付成果 ​

项目结果

交付股民保护 BI 看板，覆盖总览驾驶舱、渠道分析、转化漏斗等 3 个核心模块，梳理从公域曝光到签约转化的完整数据链路，明确资源、A 类客户、维权股、标的金额等关键口径。使用方为业务负责人和产品运营主管，替代原有手工维护报表，从需求梳理到看板上线约 2 周完成。

数据口径难点 ​

口径难点处理思路
资源数是渠道进线、咨询客户还是客服承接客户明确定义进入看板的资源口径
A 类客户符合维权条件且损失达到一定标准将业务判断转成可记录标签
维权股股票背后对应的维权标的公司与客户咨询和签约记录建立关联
风险标签来源于风控系统或业务判断与公司统一风险标签保持一致
标的金额律师用于法院材料的金额，与损失金额存在换算在看板中说明口径来源
营收估算部分阶段为预估值，不等同实际收入区分预估金额和实际金额

数据如何驱动 ​

股民保护 BI 的价值，是让业务从“报一个总数”变成“看清一条路径”。

text
渠道来源
  -> 资源质量
  -> 客户分类
  -> 签约效率
  -> 标的贡献
  -> 经营复盘

如果某个月签约下降，看板可以继续下钻：是公域曝光下降、进线资源减少、客服承接不足、A 类客户比例下降，还是某些维权标的转化变差。

成果与价值 ​

  • 梳理股民保护业务背景、核心角色、客户路径和数据口径。
  • 输出业务流程图、数据来源口径表、看板字段清单和 BI 需求文档。
  • 建立总览、渠道、运营等 dashboard demo，帮助业务快速对齐展示方向。
  • 将手工报表转化为可线上化的数据产品需求。
  • 为后续自动采集、客户标签和渠道复盘打基础。

个人能力沉淀 ​

这个项目让我再次确认，BI 看板要先理解业务路径，再把数据按路径组织起来。只有知道客户从哪里来、为什么转化、在哪里流失，看板才真正有管理价值。
